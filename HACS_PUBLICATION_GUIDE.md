## Publication Steps

### Step 1: Final Verification
Run the verification script:
```bash
./check_hacs_ready.sh
```

Expected output: "Ready for HACS publication!"

### Step 2: Create Git Repository (if not done)
```bash
git init
git add .
git commit -m "Initial release v1.0.0"
git branch -M main
git remote add origin https://github.com/soulripper13/blueprint-studio.git
git push -u origin main
```

### Step 3: Create Version Tag
```bash
git tag -a v1.0.0 -m "Release v1.0.0 - Initial stable release"
git push origin v1.0.0
```

### Step 4: Create GitHub Release
1. Go to https://github.com/soulripper13/blueprint-studio/releases
2. Click "Create a new release"
3. Select tag: `v1.0.0`
4. Release title: `Blueprint Studio v1.0.0`
5. Description (copy from CHANGELOG.md)
6. Attach `blueprint_studio.zip` (optional, will be auto-generated by workflow)
7. Click "Publish release"

### Step 5: Submit to HACS
1. Fork https://github.com/hacs/default
2. Edit `integration` file
3. Add your repository:
   ```json
   "blueprint-studio"
   ```
4. Create pull request with title: "Add Blueprint Studio"
5. Wait for HACS team review (usually 1-7 days)

**Alternative (Recommended):** Use HACS as a custom repository first
1. Open HACS in Home Assistant
2. Go to Integrations
3. Click three dots (â‹®) â†’ Custom repositories
4. Add: `https://github.com/soulripper13/blueprint-studio`
5. Category: Integration
6. Click Add

### Step 6: Test Installation
1. Install from HACS
2. Restart Home Assistant
3. Add integration from UI
4. Verify all functionality works
5. Check for any errors in logs

## Common Issues & Solutions

### Issue: HACS validation fails
**Solution:** Run `./check_hacs_ready.sh` and fix reported errors

### Issue: Integration doesn't appear in HACS
**Solution:**
1. Check repository is public
2. Verify hacs.json is valid
3. Check custom_components folder structure
4. Wait up to 24 hours for HACS index refresh

### Issue: Version mismatch warnings
**Solution:** Ensure version in manifest.json and const.py match

### Issue: Hassfest validation fails
**Solution:**
1. Check manifest.json format
2. Verify all required fields present
3. Check documentation and issue_tracker URLs are valid

## Maintenance

### For Future Releases
1. Update version in:
   - manifest.json
   - const.py
2. Update CHANGELOG.md
3. Commit changes
4. Create new tag: `git tag -a vX.Y.Z -m "Release vX.Y.Z"`
5. Push with tags: `git push origin main --tags`
6. Create GitHub release
7. HACS will auto-detect new version

### Breaking Changes
- Increment MAJOR version
- Document breaking changes in CHANGELOG.md
- Add migration guide to release notes

### Bug Fixes
- Increment PATCH version
- Document fixes in CHANGELOG.md
- Release as soon as fix is verified
